Prerequisites

  • Kubernetes cluster (1.19+)
  • kubectl configured to access your cluster
  • Helm 3 (optional, but recommended)
  • Cluster admin permissions
Note: Kube-Compactor requires read access to nodes and pods, and write access for pod evictions (in auto modes).

Option 1: Install with Helm (Recommended)

Step 1: Add the Helm repository

helm repo add kube-compactor https://kanurkarprateek.github.io/Kube-Compacter/charts
helm repo update

Step 2: Install Kube-Compactor

# Install in observe-only mode (safest for getting started)
helm install kube-compactor kube-compactor/kube-compactor \
  --namespace kube-compactor \
  --create-namespace \
  --set mode=observe

Step 3: Check installation

kubectl get pods -n kube-compactor
kubectl get optimizationconfig -n kube-compactor

Step 1: Apply CRDs

kubectl apply -f https://raw.githubusercontent.com/KanurkarPrateek/Kube-Compacter/main/k8s-controller/crds/optimizer-crds.yaml

Your First Analysis

Step 1: Wait for first analysis

The controller analyzes your cluster every 5 minutes. Wait for the first report:

kubectl get optimizationreport -n kube-compactor -w

Step 2: View optimization report

# Get latest report
REPORT=$(kubectl get optimizationreport -n kube-compactor --sort-by=.metadata.creationTimestamp -o name | tail -1)

# View the report
kubectl describe $REPORT -n kube-compactor

Step 3: Check for migration plans

# List any migration plans created
kubectl get migrationplan -n kube-compactor

# If plans exist, view details
kubectl describe migrationplan -n kube-compactor

Understanding the Output

Optimization Report Fields

  • Current Utilization: Shows CPU and memory usage across your cluster
  • Optimization Potential: Estimated savings if consolidation is performed
  • Recommended Migrations: Specific workloads that can be moved
  • Safety Score: Confidence level in the recommendation (0-100)

Migration Plan Status

  • pending - Waiting for approval or scheduled window
  • approved - Ready to execute (manual mode)
  • executing - Migration in progress
  • completed - Successfully migrated
  • failed - Migration failed (check events for details)

Next Steps

1. Enable Manual Migration Mode

Once comfortable with the recommendations:

kubectl patch optimizationconfig default-config -n kube-compactor \
  --type merge -p '{"spec":{"mode":"manual"}}'

2. Approve a Migration

# Get migration plan name
PLAN=$(kubectl get migrationplan -n kube-compactor -o name | head -1)

# Review the plan
kubectl describe $PLAN -n kube-compactor

# Approve if satisfied
kubectl patch $PLAN -n kube-compactor \
  --type merge -p '{"spec":{"approved":true}}'

3. Monitor Progress

# Watch migration progress
kubectl get events -n kube-compactor --sort-by='.lastTimestamp'

# Check controller logs
kubectl logs -f deployment/kube-compactor-controller -n kube-compactor

Configuration Options

Modes of Operation

# Observe only - no actions taken
mode: observe

# Manual approval required
mode: manual

# Auto-approve low-risk migrations
mode: semi-auto

# Fully automated (use with caution)
mode: full-auto

Optimization Strategies

# First Fit - fastest, good for homogeneous workloads
strategy: first-fit-decreasing

# Best Fit - optimal packing, recommended
strategy: best-fit-decreasing

# Network Aware - considers network traffic
strategy: network-aware

# Affinity Based - respects pod affinity rules
strategy: affinity-based

Troubleshooting

Controller not starting?

# Check RBAC permissions
kubectl auth can-i get nodes --as=system:serviceaccount:kube-compactor:kube-compactor-controller

# Check for errors
kubectl describe pod -n kube-compactor
kubectl logs -n kube-compactor deployment/kube-compactor-controller --previous

No optimization reports generated?

  • Ensure Metrics Server is installed: kubectl top nodes
  • Check controller has access to metrics: kubectl top pods -A
  • Verify optimization config exists: kubectl get optimizationconfig -A

Migrations failing?

  • Check PodDisruptionBudgets: kubectl get pdb -A
  • Verify node capacity: kubectl describe nodes
  • Review events: kubectl get events -n kube-compactor
